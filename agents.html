核心理念

Agent 不是功能，
是岗位上岗的数字员工

传统工业软件卖的是"功能模块"，Galileo OS 的 Agent 是"在岗员工"——它会感知、会判断、会调用工具、会持续学习。三个理念定义了它的与众不同。

🪞

Agent 是岗位的数字镜像

Agent 工作流

一个 Agent，是如何"上岗"的

它不是一段被动等待点击的代码，而是一个会感知、会理解、会决策、会动手、会反馈的闭环员工。五步循环，持续进化。

01

感知数据

从 IoT 设备、ERP / MES / WMS 系统、视频流、人工录入等渠道，实时接入岗位关心的全部数据。先有"看得见"，才有"管得了"。

感知 IoT 中枢 系统集成
02

理解问题

结合岗位知识与历史经验，把数据翻译成业务问题：这批订单该插队吗？这台设备快坏了吗？这个缺陷是哪道工序引入的？

理解 岗位知识 大模型推理
03

生成决策

调用排产、预测、优化等 Skill，权衡约束与目标，给出不止一个、而是"最优"的决策方案——这正是老师傅经验的数字化沉淀。

决策 经验即配置 约束求解
04

调用 Skill 执行

决策不是停在报告里。Agent 自动调用对应的 Skill：下发工单、调整参数、推送备件、触发审批——像真正的员工一样动手干活。

执行 Skill 市场 工具调用
05

反馈结果

执行结果回流，与预期比对，差异自动归因。Agent 据此修正自己的规则与模型——越用越懂你的工厂，持续进化、不会离职。
