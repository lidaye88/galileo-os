核心理念

Skill 是最小能力单元

Agent 是"数字员工"，而 Skill 是员工身上的"一项技能"。把粒度切到最细，能力才能真正可复用、可组合、可交易。

最小能力单元

一个 Skill 只解决一个明确问题：清洗一组数据、跑一次排产、读一台 PLC。粒度越小，复用越广。Agent 由若干 Skill 编排组合而成——换掉一个 Skill，Agent 的能力就随之改变。

单一职责 高复用 即插即用

可被任意对象调用

Skill 不绑定任何 Agent。它可以被 Agent 调用、被低代码应用调用、被行业解决方案批量调用、被另一个 Skill 串接调用。同一个"数据清洗"Skill，今天服务质检 Agent，明天服务能耗 Agent。

Agent 调用 应用调用 方案调用

开发者可上架分润

懂行业的老专家、独立开发者、OPC 合作伙伴，都可以把自己的经验封装成 Skill 上架到 Store。全网任意客户调用，平台自动结算分润。能力从此成为可计价的数字资产。

组合方式

从 Skill 到上线，四步搞定

不用从零造系统。先选行业模板拿到骨架，再挑 Skill 补齐能力，编排成 Agent 或应用，一键部署运行。

1

选行业模板

从行业模板库挑一个最贴近你业务的模板——水泥、化工、注塑、食品……模板自带推荐 Agent、推荐 Skill、典型指标，省去从 0 设计的功夫。

骨架 行业 Know-How 开箱即用
2

挑选 Skill

在 Skill 市场按分类浏览，把需要的能力加入清单——数据清洗、排产算法、PLC 读写、钉钉通知……每个 Skill 都有清晰的输入输出说明。

组合 按需挑选 输入输出明确
3

编排成 Agent / 应用

用可视化编排或自然语言描述，把若干 Skill 串成一条业务流——读 PLC → 异常检测 → 推送钉钉。编排结果即是一个 Agent 或一个低代码应用。

编排 可视化 / 自然语言 业务流
4

部署运行

一键部署到 Galileo OS 运行时，Skill 按调用计费，效果可观测、可回滚。验证 ROI 后再加注扩展，小步快跑。

部署 按调用计费 可观测

一个 Skill 长什么样

点击市场中任意 Skill，进入详情页都会看到如下完整信息。下面以"排产算法 Skill"为例，展示标准的 8 段详情结构。

📅
已上架 算法模型 v2.4

排产算法 Skill

综合订单、产能、约束求解最优车间排程

离散制造 流程工业 多车间协同 换模优化

基础信息

提供方：思为交互算法团队 · 分类：算法模型 · 版本：v2.4 · 更新：2026-06 · 协议：标准 Skill API · 计费：按求解次数。

能力说明

基于混合整数规划 + 启发式规则的排产求解器，支持订单优先级、设备产能、物料齐套、换模时间等多约束建模，分钟级输出可执行工单序列，并给出排产甘特图与利用率报告。

输入参数

  • 订单列表（编号、产品、数量、交期、优先级）
  • 设备产能（设备清单、可用工时、节拍）
  • 约束条件（物料、换模、班次、合批次）
  • 优化目标（最短完工 / 最低换模 / 最高利用率）

输出结果

  • 排产工单序列（工单 → 设备 → 起止时间）
  • 排产甘特图数据
  • 设备利用率 / 订单准交率报告
  • 冲突与未排原因说明

使用场景

离散制造车间日排产、多车间协同排程、紧急插单重排、换产换模优化。可被生产调度 Agent、APS 应用、行业解决方案组合调用。

组合方式

上游接 ERP 订单 Skill / WMS 齐套 Skill，下游接 MES 下发 Skill / 钉钉通知 Skill，形成"接单 → 排产 → 下发 → 通知"完整闭环。

权限与门槛

需企业版授权；调用前需完成参数 Schema 校验；首次使用建议由 OPC 顾问协助配置约束模板。
