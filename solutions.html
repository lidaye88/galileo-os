业务痛点

  • 流程：排产靠 Excel、调度靠电话，计划与执行严重脱节，插单改单牵一发动全身。
  • 数据：订单、库存、产能数据散落在多套系统，口径不一致，决策靠拍脑袋。
  • 设备：产能瓶颈说不清，哪台设备是关键约束、何时该保养全凭经验。
  • 人员：老师傅一走知识带走，新人上手慢，交接成本高、出错率高。
  • 交付：交付周期长、准时率低，客户投诉与急单成本吞噬利润。
✓

解决思路

以"数据底座 + Agent 编排 + Skill 调用 + 应用承载"的四层架构，把车间从经验驱动升级为数据驱动。

  • 采：IoT 中枢打通设备、产线、订单系统，数据实时归一。
  • 算：生产调度 Agent 调用排程算法 Skill，自动生成最优排产方案。
  • 决：异常事件触发 Agent 决策，推荐插单 / 调机 / 转产策略。
  • 执：MES 应用承接工单下发与执行反馈，闭环回到数据底座。
